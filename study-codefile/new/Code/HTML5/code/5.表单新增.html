表单属性

required > 判断输入的长度是否大于0或者输入的内容是不是null或者undefined 必填属性
autofocus > 自动获取焦点, 浏览器刷新页面有失效几率, 可以通过js中提供的focus()事件解决
autocomplete > 自动补全, 和name结合使用, input标签必需写上name名称才生效, 如果表单中需要填写比较敏感或有安全隐患的信息, 建议设置为off
maxlength > 最大输入长度
minlength > 最小输入长度
pattern > 正则表达式, 用于验证输入的内容是否符合正则表达式的规则
placeholder > 占位符, 在输入内容之前显示的内容
readonly > 只读属性, 只能读取, 不能修改
disabled > 禁用属性, 不能读取, 不能修改
value > 默认值
name > 表单名称, 用于提交表单时, 服务器端获取表单数据
form > 用于关联表单, 相当于一个袋子收集信息, 使其在不同的表单中进行数据传输
action > 指定表单提交的地址
method > 指定表单提交的方式, 默认为get
target > 指定表单提交后, 服务器端返回的数据在哪个窗口中显示
novalidate > 用于取消表单的默认验证
enctype > 用于指定表单提交的数据类型, 默认为application/x-www-form-urlencoded

新增表单类型

email 邮箱
移动端键盘抬起时，键盘中会出现 '@' 符号
输入内容会默认验证是否有 '@' 出现, 验证是不是邮箱, 不是邮箱会出现提示

这段用js阻止了默认事件, 所以没有提示

tel 手机号码格式
移动端键盘抬起时, 默认键盘只会抬起数字键盘

url 网址格式
内部会有一个验证 'http://' + 内容
在移动端中, 键盘抬起时, 键盘中会有 '.com' 按钮出现

number 数字类型
只能输入数字或者 e(无限不循环小数)
不加step属性时, 默认为整数

color 颜色选择器
返回的值是一个十六进制的字符串

search 搜索框(体现语义化)
默认有一个小样式, 搜索框后面有一个 x 符号, 点击会清空

range 自由拖动滑块
max 最大值
min 最小值
step 最小移动单位的值
value 当前值
取值范围 min~max 之间

time 小时分钟

month 年月

date 年月日

week 年 星期

datetime-local 时间
显示完整日期时间
年 / 月 / 日 - 小时 / 分钟

multiple 多选
设置当前表单允许同时选择多个文件使用 input type="file"

reset重置按钮
form表单里面输入的内容变成初始值
