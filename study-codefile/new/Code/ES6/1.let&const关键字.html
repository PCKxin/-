let关键字

let关键字用于声明变量, 与var不同的是, let声明的变量只在let命令所在的代码块内有效
就是在一个代码块内能访问,比如很适合for(let i = 0; i < 10; i++)这种循环中使用
let声明的变量不存在变量提升, 也就是说, let声明的变量一定要在声明后使用, 否则报错
比如用var的时候:
console.log(a) // undefined
var a = 1
变量a会被提升到代码块的头部, 但是用let的时候:
console.log(a) // 报错
let a = 1
因为let声明的变量不会被提升, 所以在声明前使用会报错
let声明的变量不允许重复声明, 也就是说, 在同一个作用域内, 不能用let重复声明同一个变量
let声明的变量不会成为window对象的属性, 也就是说, let声明的变量不会挂载到window对象上
window对象的属性与全局变量是等价的, 在全局作用域声明的变量, 会成为window对象的属性

const关键字

常量的意思是, 值不变的量
const关键字用于声明常量, 声明的常量的值不能改变, 一旦声明, 常量的值就不能改变
const声明的常量也是块级作用域, 也不存在变量提升, 也不会成为window对象的属性
const声明的常量必须初始化, 也就是说, const声明的常量必须赋值, 否则报错
const声明的常量不能重复声明, 也就是说, 在同一个作用域内, 不能用const重复声明同一个常量
const声明的常量不能修改, 也就是说, const声明的常量不能重新赋值, 否则报错
const声明的常量只是保证变量名指向的内存地址不变, 并不保证该内存地址的数据不变
const声明的变量在声明之前调用会报错
比如:
const a = {name: 'yuki'}
a.name = 'yuki2'
console.log(a.name) // yuki2
a = {name: 'yuki2'} // 报错
因为const声明的常量只是保证变量名指向的内存地址不变, 并不保证该内存地址的数据不变
