基础概念

背景介绍

产品三种主要形式：
Native App 原生应用
IOS: object C
Android: Java
页面存放在本地, 基于操作系统运行
优点: 能调用原声api 性能很好
缺点: 开发成本高, 发布流程繁琐, 不能跨平台

Web App 网页应用
html css js
受限于UIwebview
页面存放在服务器 基于浏览器运行
优点: 开发成本低, 发布流程简单, 能跨平台
缺点: 不能调用原声api 性能一般

Hybrid App 混合应用
object C + html + css + js
通过UIwebview加载html页面
页面可以存放在本地，也可以存放在服务器
优点: 开发成本低, 发布流程简单, 能调用原声api, 能跨平台
缺点: 性能一般

屏幕

屏幕尺寸
屏幕对角线的长度
以英寸为单位
1英寸(inch) = 2.54 厘米
常见尺寸: 3.5, 4.0, 4.7, 5.0, 5.5, 6.0, 6.5, 7.0, 7.9, 9.7, 10.1, 12.9

屏幕分辨率
屏幕水平方向和垂直方向上的像素点数
以像素px为单位
常见分辨率: 320x480, 640x960, 640x1136, 750x1334, 1080x1920, 1125x2436, 1242x2208, 1536x2048, 2048x2732

屏幕像素密度
密度越大, 屏幕越清晰
屏幕上每英寸的像素点数
以ppi为单位
计算公式: 屏幕像素密度 = (屏幕分辨率 / 屏幕尺寸的平方)
常见像素密度: 163, 326, 401, 458, 264, 326, 401, 264, 326, 264

屏幕像素比
屏幕上每个逻辑像素对应的物理像素点数
以dpr为单位
常见像素比: 1, 2, 3

视网膜屏(Retina)
也叫高清屏
像素点数是普通屏幕的2倍

像素

物理像素
也叫设备像素
以pt为单位
常见物理像素: 320, 640, 750, 1080, 1125, 1242, 1536, 2048

逻辑像素
也叫设备独立像素
以px为单位
逻辑像素 = 物理像素 * 像素比

css像素
也叫设备无关像素(device-independent)简称DIPS
主要应用在浏览器上
标准密度下, 一个css像素等于一个设备像素，1px = 1dp
2倍高清屏下, 一个css像素等于4个设备像素，1px = 4dp
以px为单位
css像素 = 逻辑像素 / 像素比

像素比
屏幕上每个逻辑像素对应的物理像素点数
以dpr为单位
devicePixelRatio = 物理像素 / 设备独立像素
可以通过window.devicePixelRatio获取

视口

布局视口
layout viewport
一般移动设备的浏览器都默认设置了一个 viewport 元标签
定义一个虚拟的布局视口, 用于解决早期的页面在手机上显示的问题。
iOS, Android 基本都将这个视口分辨率设置为 980px
所以 PC 上的网页基本能在手机上呈现，只不过元素看上去很小，一般默认可以通过手动缩放网页。
布局视口高度可以通过document.documentElement.clientHeight获取
设置布局视口:
使用meta标签
属性: name="viewport"
content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no"
width 设置布局视口的宽度 取值: 正整数 或 device-width, device-width表示布局视口的宽度等于设备独立像素
initial-scale 设置页面的初始缩放值 取值: 0.0-10.0
minimum-scale 设置页面的最小缩放值 取值: 0.0-10.0
maximum-scale 设置页面的最大缩放值 取值: 0.0-10.0
user-scalable 是否允许用户缩放页面 取值: yes/no

视觉视口
visual viewport
移动设备上浏览器可视区域的大小
视觉视口宽度可以通过window.innerWidth获取
视觉视口高度可以通过window.innerHeight获取
视觉视口宽度可以通过document.documentElement.clientWidth获取
视觉视口高度可以通过document.documentElement.clientHeight获取

理想视口
ideal viewport
布局视口的默认宽度并不是一个理想的宽度
于是 Apple 和其他浏览器厂商引入了理想视口的概念, 它对设备而言是最理想的布局视口尺寸
显示在理想视口中的网站具有最理想的宽度, 用户无需进行缩放
为了让布局视口和视觉视口的大小相同
设置理想视口:
使用meta标签
属性: name="viewport"
content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no"
